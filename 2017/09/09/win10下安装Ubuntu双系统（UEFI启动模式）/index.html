win10下安装Ubuntu双系统（UEFI启动模式）

Posted on 2017-09-09 Edited on 2019-09-08 In Linux Views:

目录

前言

由于 计算机系统 课程老师的推荐，准备尝试Ubuntu操作系统
通常，可以在虚拟机，服务器，双系统使用Ubuntu系统。虽然已经有了服务器，但是比较卡，体验效果不佳；安装虚拟机会简单一些，但是‘要玩就玩最纯正的’（我就是从这句话开始折腾的），毅然决然的选择安装双系统。

配置说明

网上的教程大致分成两种： ①传统BIOS ②uefi启动模式 前者主要是win8以前的电脑，win8以上的系统基本都是支持后者 查看，win+r,msinfo32 可查看BIOS模式 本笔记本 ： IdeaPad700，win10，支持BIOS

折腾过程

前期准备，看了不少教程，开搞！折腾过程大致如下：
第一次使用传统BIOS（lengency BIOS），按照这个教程较快的安装完成，结果开机直接进入win10。
继续百度，于是下载了一个easybcd，加入了Linux grub2的开机启动项。选择Ubuntu系统，出现下面问题： 然后又开始百度，当时也不太清楚，格式化了磁盘。现在大致了解了一点，当时安装类型选的系统共存，导致一些问题，其次便是启动模式不对。（大致这样理解）
第二次，按照这个教程，但是在磁盘分区后遇到了问题--无法卸载挂载点；卡在准备安装。在这次安装之前，还安装了一次，分区时，分出了一个biosgrub，结果安装失败，之后，改为efi系统分区。

完整的安装教程

个人大致步骤及问题: ①win10下分出一部分磁盘，使其处于未分配状态 ②制作U盘启动盘 ③进入BIOS，设置boot menu 为uefi模式，④关闭security boot (调为disable)，保存并退出 ⑤F12选择U盘启动 ⑥-1选择install Ubuntu进行安装，联网，安装类型选其它选项，进行分区，安装；如果遇到问题，比如，不能卸载以下挂载点，卡死在准备安装Ubuntu（有时确实需要等待一会），退出重进，选择试用 ⑥-2选择try，试用， 解决无法卸载挂载点：Alt+Ctrl+T，输入sudo umount -l /cdrom(无法卸载的挂载点的文件名) 解决卡在准备安装Ubuntu（主要是在试用时安装会出现一直转圈），试用界面按f,之后在右下角那里输入nonacpi 之后安装，选择联网（由于学校是校内网，拔掉网线，用热点），第三方软件及安装更新都选择，继续 ⑦分区，找到空闲， 第一，主分区，空间起始位置，交换空间，相当于虚拟内存，8192M， 第二，efi，逻辑分区，512M 第三，/home，逻辑分区，32G 第四，/usr，逻辑分区，32G 第五，/，逻辑分区，剩下所有 ⑧安装引导，选择efi所在盘符 继续 ⑨选择上海，图右边 ⑩安装，update 那个地方会安装比较久，点击文字，查看安装详情，如果在刷新就是正常 十一，重启，拔掉U盘， 进入BIOS，开启security boot，保存 选择操作系统
